游泳计时系统 通讯协议

主服务器（SwimmingScoreboard） ↔ 硬件计时控制器
版本 v2026-04  |  基于 2023-11-13 协议扩展  |  波特率 115200 8N1

1  文档目的

本文描述游泳计时系统主服务器（SwimmingScoreboard.exe）与硬件计时控制器之间的 串口 / UDP / TCP 帧级通讯协议。在 2023-11-13 版本协议基础上，加入"参数设置 / 设备状态" 双向同步所需的约定子码，作为当前实现（TimingBridge.cs 及 MainWindow.ProcessTimingDataFromHardware / SendTimingSettingsToHardware / SendDeviceStatusesToHardware）的权威说明。

2  物理层与帧结构

2.1  物理层

项参数
链路串口（推荐）/ UDP / TCP 任选其一
串口参数115200 baud, 8 data bits, no parity, 1 stop bit（115200 n 8 1）
字节序按字节顺序发送，无大小端差异；所有参数值为 8 位无符号整数
帧定界起始字节 0xF1 ('ﾑ')，结束字节 0xF4 ('ﾔ')
帧长固定 12 字节

2.2  帧结构

位置字段值 / 含义说明
D0SOH0xF1起始标识（固定）
D1'S'0x53协议识别字符（固定）
D2CMD命令字节见第 3 章"命令表"
D3命令1子码 / 附加参数设置类帧：子码；计时帧：保留（0）
D4命令2 / 泳道泳道号或参数值计时帧：0-9 终点端泳道，10-19 另一端泳道（实际泳道 = D4-10）
设置帧：参数值
D5分计时分计时帧有效；设置帧作为扩展字段（见 4.2）
D6秒计时秒计时帧有效；设置帧作为扩展字段
D71/100 秒百分之一秒计时帧有效
D8小时|毫秒(hour << 4) | ms1高 4 位 = 小时，低 4 位 = 1/1000 秒个位
D9备用0保留（固定 0）
D10备用0保留（固定 0）
D11EOT0xF4结束标识（固定）
解析要点：接收端应扫描起始字节 0xF1，检查 D1 = 0x53 且 D11 = 0xF4，不满足则丢弃该字节继续扫描；支持任意碎片累积到 12 字节后进行一次帧解析。

2.3  时间字段编码（计时帧）

以 "1 小时 23 分 45.678 秒" 为例（01:23:45.67 加 8 ms）：

字段D5 分D6 秒D7 1/100 秒D8 (小时<<4) | ms1
值 0x17 (23) 0x2D (45) 0x43 (67) 0x18 = (1<<4) | 8

主服务器解码公式：time = hour·3600 + min·60 + sec + cs/100 + ms1/1000

4  0x42 子码（参数 / 设备状态同步）

0x42 在 2023-11-13 原始协议中定义为 "设置命令" 的通用外壳， 其 D3 为 "命令1"、D4 为 "命令2"。主服务器与硬件计时控制器在此外壳上 商定了如下子码，用于实现"参数设置"与"设备状态"的双向同步。

4.1  单参数子码（D3 = 0x01 ~ 0x07，D4 为值）

D3设置项 D4 取值单位有效范围示例
0x01LaneCloseTime  (泳道关闭时间) 秒0 ~ 255D4 = 20 → 20 秒
0x02StartBlockCloseDelay  (出发台关闭延迟) 0.1 秒单位0 ~ 255 → 0 ~ 25.5 秒D4 = 30 → 3.0 秒
0x03ResultConfirmCloseDelay  (成绩确认关闭延迟) 0.1 秒单位0 ~ 255D4 = 30 → 3.0 秒
0x04FalseStartThreshold  (抢跳判定阈值) 0.01 秒单位0 ~ 100 → 0 ~ 1.00 秒D4 = 10 → 0.10 秒
0x05SplitDisplayTime  (分段成绩显示时长) 秒0 ~ 255D4 = 5 → 5 秒
0x06FirstPlaceHoldTime  (第 1 名成绩停留时间) 秒0 ~ 255D4 = 3 → 3 秒
0x07FinishPosition  (终点位置) 枚举0 = 左端，1 = 右端D4 = 0 → 左端
说明：任意一方（主服务器或硬件）修改上述参数后，应以单帧形式把该子码发送给对端； 对端收到后更新本地状态并向其管辖的所有下游客户端（EXE/HTML 计时控制 或 连接中的显示屏等）进行广播。

4.2  比赛距离子码（D3 = 0x20）

字段含义编码
D3子码0x20（固定）
D4趟数 (totalLaps)1 ~ 255
D5左端触板总次数0 ~ 255
D6右端触板总次数0 ~ 255
D7距离高字节距离米数 / 256
D8距离低字节距离米数 % 256（D7:D8 合成 16 位米数，最大 65535）
触发时机（主服务器 → 硬件）：
  • 比赛控制台选中新项目 / 新组次（SetCurrentEvent, SetCurrentHeat）
  • 硬件计时控制器首次连接
左/右触板次数根据当前FinishPosition 与当前项目趟数计算。以 50 米池 / 出发在左端为例：
项目趟数左端(出发)右端(到达)
50 米101
100 米211
200 米422
400 米844
1500 米301515

4.3  设备状态子码（D3 = 0x10）

字段含义编码
D3子码0x10（固定）
D4泳道号0 ~ 9
D5左端设备损坏位图 位图（1 = 损坏 / 0 = 正常）：
bit4bit3bit2bit1bit0
出发台盲表 3盲表 2盲表 1触板
D6右端设备损坏位图同上（bit0 触板 / bit1 盲1 / bit2 盲2 / bit3 盲3 / bit4 出发台）
一次全量同步："设备状态管理"对话框确认后，主服务器会按 0 ~ 9 道顺序发送 10 帧 0x42 / 0x10；硬件端也可按任意顺序随时发送 0~9 道中任何一帧，接收方仅对收到的那一道更新状态。

5  参数双向同步时序

5.1  主服务器 → 硬件（下行）

触发时机：

  • 主界面"参数设置"对话框点击"确定"
  • 来自 EXE/Web 比赛控制端的 WebSocket 指令 SET_LANE_CLOSE_SETTINGS
  • "设备状态管理"对话框点击"确定"
  • 单点设备状态变更（远程 SET_DEVICE_STATUS）
  • "赛事信息"中修改泳道数 / 泳池长度后保存
  • 比赛控制台切换项目 / 组次（SetCurrentEvent / SetCurrentHeat）→ 发送 0x42 / 0x20 比赛距离帧
  • 硬件连接刚刚建立（以服务器当前参数为准做一次全量下发，包含 0x40 + 7 个 0x42 参数 + 10 个 0x42/0x10 设备状态 + 0x42/0x20 比赛距离）

下发顺序（SendTimingSettingsToHardware + SendDeviceStatusesToHardware）：

0x40   D3=Lanes    D4=PoolLen
0x42   D3=0x01     D4=LaneCloseTime (s)
0x42   D3=0x02     D4=StartBlockCloseDelay×10
0x42   D3=0x03     D4=ResultConfirmCloseDelay×10
0x42   D3=0x04     D4=FalseStartThreshold×100
0x42   D3=0x05     D4=SplitDisplayTime (s)
0x42   D3=0x06     D4=FirstPlaceHoldTime (s)
0x42   D3=0x07     D4=FinishPosition (0|1)
0x42   D3=0x10     D4=0   D5=left_mask   D6=right_mask
0x42   D3=0x10     D4=1   D5=left_mask   D6=right_mask
...
0x42   D3=0x10     D4=9   D5=left_mask   D6=right_mask

5.2  硬件 → 主服务器（上行）

主服务器识别 0x40 / 0x41 / 0x42 后，进入 ApplyHardwareSettingFrame 流程：

  1. 根据 CMD 与 D3 解码参数值；
  2. 更新 _poolConfig / _laneCloseSettings / _laneDeviceStates；
  3. 调用 SaveTimingSettings() + AutoSaveData() 持久化；
  4. 调用 Broadcast()，把新状态同步到全部 EXE / HTML / 显示 / 计时 Web 等 WebSocket 客户端；
  5. 为避免回环，_applyingHardwareSettings 在本次处理期间为 true， 抑制 SendTimingSettingsToHardware / SendDeviceStatusesToHardware 重新下发。

5.3  时序图

① EXE 本地参数设置确认 用户 → SwimmingScoreboard
主服 → 硬件：0x40 + 七个 0x42/0x00~0x07
主服 → Web/显示：Broadcast
② 硬件端修改参数 硬件 → SwimmingScoreboard：一到多帧
主服 → EXE/Web/显示：Broadcast
主服不回送硬件（回环抑制）
③ 远程客户端修改参数 EXE/Web → SwimmingScoreboard（WebSocket）
主服 → 硬件：同 ①
主服 → 其它客户端：Broadcast

7  帧示例

7.1  设置泳池：10 道 / 50 米

F1 53 40 0A 32 00 00 00 00 00 00 F4
  SOH 'S' CMD D3  D4  D5 D6 D7 D8 D9 D10 EOT

7.2  设置抢跳阈值 0.10 秒

F1 53 42 04 0A 00 00 00 00 00 00 F4   // D3=0x04, D4=10 → 0.10 秒

7.3  设置第 3 道左端"触板 + 盲表 1"损坏，右端全部正常

F1 53 42 10 03 03 00 00 00 00 00 F4
                // D3=0x10 D4=3 (道号)
                // D5=0x03 = 0b00011 (左端: 触板+盲1)
                // D6=0x00 (右端: 全部正常)

7.4  比赛距离：400 米 / 8 趟 / 左端 4 次 右端 4 次

F1 53 42 20 08 04 04 01 90 00 00 F4
           20  .  08  04 04  .  01 90    .
           子码0x20 . D4=8 D5=4 D6=4 . D7:D8=0x0190 (400 米)

7.5  触板：第 5 道终点端，成绩 27.83 秒

F1 53 16 00 05 00 1B 53 00 00 00 F4
        16  .  05  .  1B 53  .
        触板 .  D4=5(终点端) . Min=0 Sec=27 Cs=83

8  版本与兼容

  • 2023-11-13：奠定 12 字节帧、0x16~0x21/0x7F、0x40/0x41/0x42 外壳。
  • 2026-04：在 0x42 下约定子码 0x01~0x07（参数）及 0x10（设备状态）， 新增双向同步实现；帧长 / 起止 / 命令字节码保持完全兼容。
  • 2026-04-b：追加 0x42 / 0x20 子码（比赛距离 + 左右触板总次数 + 距离米数 16 位）， 切换项目/组次时自动下发，使硬件可预知当前项目的触板总数。

任何旧版硬件只处理 0x16~0x21/0x7F/0x40/0x41 仍可正常收发； 收到它无法识别的 0x42 子码时应静默忽略而非错误重置。
