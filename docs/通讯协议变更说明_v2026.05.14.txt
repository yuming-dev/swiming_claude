  游泳计时通讯协议变更说明  v2026.05.14
  （基于"游泳计时通讯协议 2023-11-13" 的增量修改）

帧格式（不变）：
    F1 53 [CMD] [D3] [D4] [D5] [D6] [D7] [D8] [D9] [D10] F4
    共 12 字节，D0=0xF1（SOH），D1=0x53（'S'），D2=命令，D11=0xF4（EOT）


┌──────────────────────────────────────────────────────────────┐
│ A. 新增命令                                                   │
└──────────────────────────────────────────────────────────────┘

─────────────────────────────────────────────────────────────────
A1. 0x39  Set_DateTime         方向：PC → 硬件
─────────────────────────────────────────────────────────────────
触发：连接建立后 PC 主动一次性下发，让硬件 RTC 对齐计算机时钟。

字段：
    D3  年低字节         year & 0xFF
    D4  年高字节         (year >> 8) & 0xFF
    D5  月               1 – 12
    D6  日               1 – 31
    D7  时               0 – 23
    D8  分               0 – 59
    D9  秒               0 – 59
    D10 0

示例：2026-05-13 14:30:45 → 0x07EA = 2026
    F1 53 39 EA 07 05 0D 0E 1E 2D 00 F4


─────────────────────────────────────────────────────────────────
A2. 0x3A  Set_PoolSingleOrDoubleTP    方向：PC → 硬件
─────────────────────────────────────────────────────────────────
触发：用户切换"单边 / 两端安装"时 + 连接建立后同步当前配置。
硬件需将本设置持久化到 SD 卡。

字段：
    D3  0 = 两端安装；1 = 单边安装
    D4–D10  保留 0


─────────────────────────────────────────────────────────────────
A3. 0x4B  BatteryVoltage        方向：硬件 → PC（主动上报）
─────────────────────────────────────────────────────────────────
触发：硬件周期性上报（建议每 5–10 秒，或电压跳变超过阈值时）。

字段：
    D3  电压 mV 高字节
    D4  电压 mV 低字节
    D5–D10  0

PC 端：
    mV = (D3 << 8) | D4
    V  = mV / 1000.0
    显示阈值（2026-05-13(2) 按硬件实际电池 7.4V 标称 / 2节锂电串联包定值）：
        <  7.00 V          →  红色（严重低电，请立即更换/充电；接近耗尽截止 6.0V）
        7.00 V – 7.40 V    →  琥珀色（偏低，注意监视）
        >= 7.40 V          →  绿色（健康；标称 7.4V，满充 8.4V）

示例：7.85 V → mV = 7850 = 0x1EAA
    F1 53 4B 1E AA 00 00 00 00 00 00 F4


─────────────────────────────────────────────────────────────────
A4. 0x4C  Set_ForceAllOpen      方向：PC → 硬件
─────────────────────────────────────────────────────────────────
触发：操作员点击"全开 / 关闭"功能按钮。

字段：
    D3  泳道号
            0xFF      = 全部泳道
            0x00–0x09 = 指定单道
    D4  1 = 进入"全开"；0 = 恢复正常状态机
    D5–D10  0

语义：
    全开：硬件无视触板/盲表/出发台内部 Closed/Touched 状态机，
          任意信号都按"该端处于 Open"上报（仍走原 0x16/0x17/0x18/
          0x19/0x1A 帧，只是不再被自身状态过滤）。
    关闭：恢复正常流程（已 Touched 的设备不重复响应、Closed 不响应等）。

建议：进入"全开"模式时，硬件点亮一个 LED 或屏幕加显标识，
      让操作员能直观看到"特殊模式"在生效。


─────────────────────────────────────────────────────────────────
A5. 0x47  Set_LaneOpenClose      方向：PC → 硬件   【2026-05-14 新增】
─────────────────────────────────────────────────────────────────
触发：比赛控制 → 泳道控制 → "全部打开 / 全部关闭 / 道次打开 / 道次关闭"
      4 个按钮中的任一个。

字段：
    D3  泳道号
            0xFF      = 全部泳道
            0x00–0x09 = 指定单道
    D4  1 = 打开（该道纳入比赛，触板/出发台/盲表按正常状态机响应）
        0 = 关闭（该道完全屏蔽，硬件不接受其任何信号，相当于空泳道）
    D5–D10  0

硬件 dispatcher 约定：on-the-wire D2 = 0x47 → hardware case = 0x47 - 0x10 = 0x37
                    所以 USER/swimtime.h 应定义：
                        #define Set_LaneOpenClose    0x37

示例帧：
    全部道打开:   F1 53 47 FF 01 00 00 00 00 00 00 F4
    全部道关闭:   F1 53 47 FF 00 00 00 00 00 00 00 F4
    第 4 道打开:  F1 53 47 04 01 00 00 00 00 00 00 F4
    第 4 道关闭:  F1 53 47 04 00 00 00 00 00 00 00 F4

与 0x4C 的语义区分：
    0x47 道次开/关 → 泳道整体启用/禁用（关闭后该道相当于空泳道）
    0x4C 全开/恢复 → 设备状态机覆盖（接受所有信号 vs 按状态过滤）
    两者独立。0x47 = 关 优先级最高，被关闭的道无视 0x4C = 全开 也不上报。

─────────────────────────────────────────────────────────────────
B1. 0x1A  StartingBlock — D10 新增"符号位"
            方向：硬件 → PC
─────────────────────────────────────────────────────────────────
出发反应时数据帧。

字段（其它不变，新增 D10）：
    D3  命令子码（同原协议）
    D4  泳道号（0–9 终点端 / 10–19 另一端）
    D5  分
    D6  秒
    D7  1/100 秒
    D8  (小时 << 4) | (1/1000 秒)
    D9  0
    D10  ★ 0 = 正常出发；非 0 = 抢跳 / 犯规

PC 端 D10 ≠ 0 时把时间取负显示（例如 -0.020 s）。


─────────────────────────────────────────────────────────────────
B2. 0x41  Set_ArmDelay_Time — D5 含义明确
            方向：PC → 硬件
─────────────────────────────────────────────────────────────────
原帧结构不变，但 D5 字节过去软件写死 50，现在改为可配置的
"盲表代替成绩延迟时间"。硬件本来就支持该字节，无协议改动。

字段：
    D3  触板正常关闭时间                秒
    D4  触板抢跳后关闭时间              秒
    D5  ★ 盲表代替成绩延迟时间          0.1 秒（50 = 5.0 秒）
    D6  成绩显示时间                    秒
    D7  出发信号边沿（0 = 下降沿）      bool
    D8  道次 + 发令位混合字节           bit
    D9  触板关闭延迟                    秒
    D10 出发台关闭延迟                  秒


─────────────────────────────────────────────────────────────────
B3. 0x42  SetCommand — 新增子码 0x09
            方向：双向（PC 下发 / 硬件回传）
─────────────────────────────────────────────────────────────────
0x42 是单项参数读写命令，D3 为子码。

子码总览：
    D3      参数                          D4 单位       状态
    ────  ──────────────────────────────  ───────────  ─────
    0x01    LaneCloseTime                 秒           既有
    0x02    StartBlockCloseDelay          0.1 秒       既有
    0x03    ResultConfirmCloseDelay       0.1 秒       既有
    0x04    FalseStartThreshold           0.01 秒      既有
    0x05    SplitDisplayTime              秒           既有
    0x06    FirstPlaceHoldTime            秒           既有
    0x07    FinishPosition (0=左,1=右)    —            既有
    0x08    BlindWatchCount               —            既有
              （高 4 位 = 左 1–3，低 4 位 = 右 1–3）
    0x09 ★  BlindReplaceDelay             0.1 秒      新增
              （盲表代替成绩延迟）


─────────────────────────────────────────────────────────────────
B4. 0x43  Set_MatchEvent — D8 新增"接力标志"
            方向：PC → 硬件
─────────────────────────────────────────────────────────────────
触发：每次进入新组比赛前下发。

字段（其它不变，新增 D8）：
    D3  总圈数
    D4  右端期望触板次数
    D5  左端期望触板次数
    D6  泳道开关位图 0–4 道（bit = 1 → 有运动员）
    D7  泳道开关位图 5–9 道
    D8  ★ 0 = 个人项目；1 = 接力项目
    D9  0
    D10 0

硬件按 D8 决策：
    D8 = 1  →  为每棒分别测量接力反应时（运动员从出发台离台
              到上一棒触板的时间差），通过 0x1A 上报，可能多次
    D8 = 0  →  只测一次起跳反应时


─────────────────────────────────────────────────────────────────
B5. 0x46  Set_TPSBMB_State — 语义澄清
            方向：PC → 硬件
─────────────────────────────────────────────────────────────────
帧格式不变（20 位 TP / 20 位 SB / 20 位 MB 位图）。

★ 语义补充：bit = 0 表示"硬件无此设备"，原因可以是
    · 损坏（Broken）
    · 或 未安装（NotInstalled，例如单边模式下右端整列）

硬件收到 bit = 0 即停止期待该端信号，不必区分这两种原因。

    命令                              方向          触发时机
    ────────────────────────────────  ───────────  ────────────────────
    0x39  Set_DateTime                PC → 硬件     连接建立后一次
    0x3A  Set_PoolSingleOrDoubleTP    PC → 硬件     模式切换 + 连接建立

    0x46  Set_TPSBMB_State(语义)      PC → 硬件     模式切换 / 设备状态变更
    0x4B  BatteryVoltage              硬件 → PC     周期性主动上报
    0x4C  Set_ForceAllOpen            PC → 硬件     用户点击全开/关闭按钮
    0x47  Set_LaneOpenClose           PC → 硬件     比赛控制 → 泳道控制 4 按钮

┌──────────────────────────────────────────────────────────────┐
│ D. 兼容性建议                                                 │
└──────────────────────────────────────────────────────────────┘

· 硬件解析时遇到未识别的 D8/D9/D10 扩展字节，应忽略而不丢帧。
· 0x4B 电压上报建议带"上一次发送的时间戳"内部判断，避免短时间
  内重复刷屏。
· 0x4C 全开命令进入后，硬件点亮 LED 或屏幕加显标识，让操作员
  看到"特殊模式"在生效。
· 0x39 校时建议硬件收到后用 0x42 子码（自定义子码或日志通道）
  回传一次时间，方便调试。
