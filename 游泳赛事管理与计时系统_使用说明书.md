## 1. 系统概述

### 1.1 设计目标

本系统是一套**完整的游泳比赛管理与计时解决方案**，覆盖赛事全生命周期：

- **赛前**：赛事创建 / 运动员报名 / 自动或手动分组 / 秩序册编辑 / 纪录数据导入
- **赛中**：硬件计时实时联动 / 三端控制台同步 / 设备状态管理 / 大屏展示 / 媒体播放
- **赛后**：成绩确认与排名 / 多轮晋级 / 成绩册编辑 / 多格式文档输出（PDF / DOC / HTML）

系统遵循 **World Aquatics（FINA）SW 3 竞赛规则** 自动分组（蛇形/种子排位、矛头泳道分配等）。

### 1.2 系统组成

| 组件 | 说明 | 运行方式 |
|---|---|---|
| **主服务器** `SwimmingScoreboard.exe` | 赛事管理核心，连接计时硬件，承载所有数据 | Windows 应用程序 |
| **远程计时控制台** `RemoteTimingControl.exe` | 独立 PC 上的比赛控制客户端 | Windows 应用程序 |
| **HTML 计时控制台** `race_control.html` | 浏览器端比赛控制（与 EXE 等价） | 浏览器全屏 |
| **网页检录台** `checkin.html` | 检录台核对身份、登记 DSQ/DNS/DNF | 浏览器 |
| **大屏显示** `display.html` | 观众大屏（项目/成绩/纪录/媒体）| 浏览器全屏 |
| **排名屏** `leaderboard.html` | 项目/团体/纪录排名展示 | 浏览器全屏 |
| **显示控制** `control.html` | 大屏显示模式遥控 | 浏览器 |
| **在线报名** `register.html` | 运动员/接力队自助报名 | 浏览器 |
| **EXE 注册工具** `RegistrationTool.exe` | 桌面端运动员注册客户端 | Windows 应用程序 |
| **自动化测试机器人** `test_bot.html` | 模拟报名/比赛流程，便于演示与压测 | 浏览器 |
| **计时模拟器** `TimingSimulator.exe` | 在没有实物硬件时调试通讯协议 | Windows 应用程序 |
| **硬件参数调试机器人** `ParamDebugBot.exe` | 与硬件计时器双向调试 | Windows 应用程序 |

### 1.3 系统要求

- **操作系统**：Windows 10 / 11（推荐 64 位）
- **运行环境**：.NET Framework 4.5+（系统通常已预装）
- **网络**：局域网即可（主服务器与各客户端、大屏在同一子网）
- **端口**：`3002`（WebSocket）
- **浏览器**：Chrome / Edge（推荐最新版）
- **硬件计时器**：通过串口 / UDP / TCP 任一方式连接（协议详见 §3）

## 2. 安装与启动

### 2.1 安装

1. 解压 `游泳赛事管理系统_安装包_<日期>.zip`
2. 双击 `Setup.exe`，选择安装目录（默认 `C:\SwimmingTimingSystem`）
3. 安装完成，桌面 / 开始菜单生成快捷方式（主服务器、注册工具、远程计时台）

### 2.2 启动主服务器

1. 双击 **"游泳赛事管理主服务器"**（`SwimmingScoreboard.exe`）
2. 首次启动会要求设置管理员密码
3. 登录后进入主界面（多 Tab：系统工作状态 / 赛事管理与报名 / 比赛控制 / 成绩与排名 / 纪录管理 / 比赛类型管理 / 文档编辑 / 系统日志）

### 2.3 启动远程计时控制台（异机操作）

1. 双击 **"远程计时控制台"**（`RemoteTimingControl.exe`）
2. 顶部输入主服务器地址 `<IP>:3002`
3. 点击 **连接**

### 2.4 打开 HTML 控制台 / 大屏 / 注册端

- 浏览器打开如下页面之一（首次启动会显示**全屏连接遮罩**，输入主服务器 IP 并点击"连接"按钮即可）：
  - 计时控制台 `race_control.html`
  - 大屏显示 `display.html`
  - 排名屏 `leaderboard.html`
  - 显示控制 `control.html`
  - 在线报名 `register.html`
  - 检录台 `checkin.html`

### 2.5 防火墙

请允许 `SwimmingScoreboard.exe` 通过 Windows 防火墙（专用 / 公用都勾选），否则其它客户端将无法连接 3002 端口。

## 3. 系统架构与通讯

```
            ┌──────────────────────────────┐
            │        计时硬件控制器         │
            │   (串口 / UDP / TCP)         │
            └──────────┬───────────────────┘
                       │ 12 字节帧 F1 53 ... F4
                       │（双向同步：参数 / 设备 / 比赛距离）
        ┌──────────────▼──────────────────┐
        │   主服务器 SwimmingScoreboard    │
        │   (WebSocket :3002 / JSON 持久化) │
        └──────────────┬──────────────────┘
                       │
        ┌──────────────┼─────────────────┬──────────────┐
        ▼              ▼                 ▼              ▼
   EXE 远程台    HTML 计时台      大屏 / 排名屏     在线报名 / 检录台
```

### 3.1 三个计时控制台等价

主服务器 / `RemoteTimingControl.exe` / `race_control.html` **功能完全等价**——任何一端按下就位 / 发令 / 确认成绩 / 计时器清零，都通过 WebSocket 广播到其它两端 + 硬件，状态保持一致。

### 3.2 硬件计时协议（摘要）

- 12 字节帧 `F1 53 [子码] [数据 ×8] F4`
- `0x42` 子码扩展（双向）：
  - `0x01-0x07`：泳道关闭时间 / 出发台延迟 / 抢跳阈值 / 终点位置
  - `0x08`：左右盲表数量（高 4 位左 / 低 4 位右）
  - `0x10`：设备好/坏位图
  - `0x20`：比赛距离 + 左右触板次数
- 详见 `游泳计时系统通讯协议.pdf`

## 4. 比赛类型管理

`比赛类型管理` Tab 用于配置 5 种基础维度，**任意修改都会即时同步到所有 WPF 下拉、网页报名端、大屏和远程台**（通过 `NotifyMetadataChanged`）：

| 维度 | 用途 | 默认值 |
|---|---|---|
| **比赛项目** | 报名 / 赛程的可选项目（50/100/200米…）| 标准游泳项目 17 项 |
| **组别** | 报名时分类（甲/乙/丙/丁/戊/己/青少年/少年/青年/成年/大师/老年）| 6 个标准组 |
| **性别** | 男 / 女 / 混合 | 三个 |
| **赛次** | 预赛 / 半决赛 / 决赛 / 自定义 | 三个 |
| **组数** | 1组 / 2组 / … 的预设值 | 1-8 |

每个维度都有 **编辑 / 导入CSV / 导出CSV / 下载模板** 4 个按钮：

- 编辑器：弹窗 DataGrid，左侧自动序号 1, 2, 3 …，可增/删行；点击 **确认保存** 立即生效
- 导入 / 导出 CSV 与 Excel/WPS 兼容（UTF-8 BOM）

> ⚠ 组别现为人工分类（如奥运/全运的甲乙丙组），与年龄无关；不再像旧版按 MinAge/MaxAge 自动判定。

## 6. 运动员管理

### 6.1 工具栏布局

第 1 行：**添加 / 修改选中 / 删除选中** ｜ **导入CSV / 导出CSV / 导出报名模板** ｜ **号码区间设置**
第 2 行：项目 / 组别 / 性别 / 姓名（模糊）/ 编号（模糊）/ 重置筛选

### 6.2 添加 / 修改 / 删除

- **添加运动员**：弹出对话框，可一次性填齐：姓名 / 性别 / 出生日期 / 身份证 / 代表队 / 单位简称 / 组别 / 联系电话 / 协会注册号 / 备注 / 报名项目 / 报名成绩 等
- **修改选中** / **删除选中**：支持多选（Ctrl/Shift），按 Delete 键也可删除

### 6.3 单位简称

新增列 `单位简称`（CountryShort），用于大屏 / 成绩册简显（如 国队队伍名"广东省体育代表团"显示为"粤"）。

### 6.4 接力队员条目

接力队报名后会自动生成 4 位 **接力队员个人条目**（Notes 以 `接力队员` 开头），它们：
- 在"运动员管理"中**自动隐藏**，避免界面混乱
- 在"接力队管理"中按队伍归类显示
- 在"导出 CSV"时**包含**进去（便于导出全量名单）

### 6.5 号码区间设置

按代表队设置号码段（如 北京队 001-050，江苏队 051-100）。新注册运动员自动在该队号码段内取号；也可手动指定。

### 6.6 CSV 表头格式

`号码 , 姓名 , 性别 , 代表队 , 项目 , 报名成绩 , 年龄 , 出生日期 , 身份证号 , 电话 , 备注 , 组别 , 单位简称`

- **导出 CSV**：当前所有运动员（含接力代表与接力队员）
- **导出报名模板**：空白模板 + 表头 + 2 条示例行；用 Excel/WPS 编辑后用"导入CSV"回灌
- **导入 CSV**：号码列留空时由系统按号码段自动分配

## 7. 接力队管理

`接力队管理` Tab：按"性别+项目"分组展示所有接力队，每队列出 4 棒队员。

### 7.1 添加 / 删除接力队

- **添加接力队**：选项目 / 性别 / 队名 / 报名成绩 / 4 棒队员姓名（可留空，后续补）
- **删除选中队伍**：同时清除 _swimmers 中的代表条目和 4 位队员条目

### 7.2 防重复登记（v2026.04 修复）

接力队以 (队名 + 性别 + 项目) 为唯一键。重复提交同一队伍时**不会新增重复条目**，而是用最新数据覆盖现有队伍（更新报名成绩与棒次）。

### 7.3 同步分组信息

按下"**同步分组信息**"按钮，把代表条目的 Heat/Lane 信息同步到 4 棒队员。这样接力队员个人条目（用于检录核对身份）也带组别 / 道次。

## 8. 赛程与分组

### 8.1 一键生成日程

点击 **一键生成日程**，弹窗让用户为所有项目选择赛次方案：

- 只决赛（推荐，适合中小型赛事）
- 预赛 → 决赛
- 预赛 → 半决赛 → 决赛

系统按"组别 → 性别 → 项目 → 赛次"维度合成场次，按距离 / 项目类型自动排序到上午（预赛）/ 下午（半决赛）/ 晚上（决赛）三段：

| 时段 | 起始时间 |
|---|---|
| 上午 | 09:00 |
| 下午 | 14:00 |
| 晚上 | **17:30**（v2026.04 由 19:00 改为 17:30）|

### 8.2 预赛自动分组（FINA SW 3 规则）

按 **World Aquatics SW 3.1.1.4 / 3.1.1.5 / 3.1.2** 标准实现：

#### 短距离项目（50 / 100 / 200 米）

- `seedHeats = min(总组数, 3)` ：**最后 3 组**进行循环（种子）排位
- 前面组进行直接排位，由慢到快编号；最慢组（第 1 组）拿剩余不满人数
- 种子分配：第1名→最后组中道，第2名→倒数第2组中道…第4名→最后组次中道，依次循环

#### 中长距离 / 接力（400 / 800 / 1500 米 / 4×N 接力）

- `seedHeats = min(总组数, 2)` ：**最后 2 组**进行循环（种子）排位

#### 矛头泳道（Spearhead）分配

| 泳池 | 优先级（最快→最慢）|
|---|---|
| 8 道 | `4, 5, 3, 6, 2, 7, 1, 8` |
| 10 道 | `4, 5, 3, 6, 2, 7, 1, 8, 0, 9` |

成绩最快者必占 4 道；最慢者排到边道（10 道场景下 0/9 道）。

> 注：8 人以下单组直接决赛时，0、9 道空着不用，FINA 标准如此。

### 8.3 出场编排微调

`出场编排微调` Tab 的 DataGrid 可对每位运动员的"组次/道次"做精细调整。支持：
- **手动均分**：输入"每组 N 人"自动分组
- **同组互换泳道**
- **追加分组**：把新报名的运动员塞到最后一组空位 + 必要时新建组

### 8.4 晋级处理

`成绩与排名` Tab 的 **晋级处理** 按钮 → 弹出 `PromotionQueryWindow`：

- 查询当前赛次的总排名（不分组统一排名）
- 设置晋级人数（默认：预赛→半决赛取 16，半决赛→决赛取 8）
- 一键执行晋级 + 自动分组到下一赛次

每组确认成绩后系统也会**主动询问**是否进入晋级流程（仅当全部组完赛）。

## 9. 比赛控制

`比赛控制` Tab 三栏布局：

```
左：赛程导航 + 计时源对比          中：泳道实时状态                 右：比赛控制按钮
```

### 9.1 赛程导航（含组别 / 已完赛标记）

树形结构按"单元 → 项目 → 组次"展开：

```
第1单元（2026-05-01上午）
├ [甲组] 男 50米自由泳 预赛
│   ├ 第1组 [已完赛]
│   ├ 第2组 [已完赛]
│   └ 第3组 (共3组)   ← 当前组高亮
├ [乙组] 男 100米自由泳 预赛
...
```

- 项目标签自动加 `[组别]` 前缀
- 组次确认成绩后立即追加 `[已完赛]` 灰色标记
- 当前选中组高亮蓝色

> ✅ **已确认成绩永久锁定**：通过 `_confirmedHeats` 集合记录每个已确认组次（key = 组别|性别|项目|赛次|组次），并入档持久化；后续任何数据变化都不会让"[已完赛]"标记消失。

### 9.2 顶部当前比赛行

显示 `[组别] 性别子 项目 赛次 第N/M组`，便于操作员一眼确认当前组别。

### 9.3 泳道实时状态

每条泳道一行，左右对称布局：

```
[T] 盲3 盲2 盲1 出发 触板 [圈 ▲▼] │ 姓名+进度 │ [▲▼ 圈] 触板 出发 盲1 盲2 盲3 [T]
```

- **左右盲表数量**：可独立设置 1-3，超出数量的圆点占位但隐藏（盲1 / 出发 / 触板的位置始终固定）
- **T 按钮**：手动触板，左右各一，可独立启用 / 禁用

- **接力项目**：第 1 行显示代表队（如"江苏"），第 2 行显示 `第N棒: 队员姓名`（按当前圈数自动推断）
- **空泳道**：整行淡化（透明度 35%），名字显示"（空泳道）"
- **圈数 ▲▼ 微调（v2026.05 新增）**：每道左 / 右各有一对 spinner，按哪一侧只调整哪一侧的剩余圈数显示。值钳到 `[0, 该侧实际触板次数]`：50m 出发端 0 / 到达端 1；100m 双侧各 1；150m 出发端 1 / 到达端 2；200m 双侧各 2；以此类推。三端（主控 / EXE / HTML）操作一致，结果由服务器统一计算后实时广播。
- **反应时颜色（v2026.05 新增）**：硬件检测到反应时低于阈值时，反应时数字变 **红色** 提醒裁判；行边框琥珀色提示"可疑起跳"。**不再自动 DSQ**——是否判罚由裁判手动按 DSQ 按钮决定。
- **备注栏（v2026.05 新增）**：DSQ/DNS/DNF 优先红字显示；否则显示破/平纪录标识（如 `WR` / `=AR` / `WR/NR`），金色高亮（详见 §11）。

### 9.5 显示控制按钮组（向大屏推送）

- **比赛视图 / 出发表 / 组成绩 / 总排名 / 团体积分 / 颁奖典礼 / 纪录榜 / 赛程表 / 晋级名单 / 欢迎画面 / 暂停画面**

- **图片 / 视频**：上传图片或视频全屏播放

### 9.6 计时器清零的两种语义（v2026.04 重构）

按"当前组是否已确认成绩"自动选择行为：

#### A) 已确认成绩的组（赛程导航已显示 [已完赛]）

- **彻底清零复位**，准备进入下一组比赛
- **已确认成绩永久保留**（在 `_confirmedHeats` + 各运动员 `Results` 中已落盘）
- 自动 `SetCurrentHeat(N+1)` 切到下一组
- 若已是本赛次最后一组：泳道行整体清空（"（空泳道）"），顶部"组别"行改为"第 N 组已完赛 / 共 M 组"，**大屏切到欢迎画面**

#### B) 未确认成绩的组（如抢跳召回重赛）

- 只清当前组的时间 / 分段数据
- **DSQ / DNS / DNF 等备注状态保留**
- 停留在当前组，重新发令

### 9.7 切组守卫

当前组未确认成绩时，禁止切换到其它项目 / 组次（保护操作员避免误操作丢成绩）。三个计时控制台（主控 / EXE / HTML）规则一致。

### 9.8 已确认组防再次开赛

`就位 / READY` 按钮：当前组已确认时，弹"操作被阻止"提示，必须切到下一组才能继续。

### 9.9 起跳犯规处理（v2026.05 改为人工判罚）

| 步骤 | 系统行为 |
|---|---|
| 硬件检测到反应时 < `抢跳判定阈值` | 仅置 `IsSuspectFalseStart=true`，反应时数字变红、行边框 amber 提示 |
| 三端日志 | 输出 `⚠ 起跳可疑（待裁判确认）泳道X 反应时:0.082s` |
| 裁判判罚 | 在该泳道按 **DSQ** 按钮（三端均可）；备注栏显示 `DSQ`、运动员状态置 DSQ |
| 裁判不判罚 | 反应时仍标红，但成绩照常计入；后续可继续比赛 |
| 实时上大屏 | 任何手动 DSQ 操作通过 `MARK_DSQ` 立刻广播到 `display.html`，备注栏立即出现 `DSQ` |

> ⚠ 之前版本"反应时低于阈值自动 DSQ + 在确认成绩时把抢跳标 DSQ"两条逻辑均已删除；判罚必须**人工确认**。

### 11.3 自动比对工作流（v2026.05 新增）

```
触板完赛  → CheckRecords() → 写 result.RecordNote → 广播 + UI/打印 显示
                                    ↓ 裁判 ConfirmResult
                              UpdateRecordsAfterConfirm() → 刷新 SwimmingRecord
                                    ↓ AutoSaveData
                              入档持久化（CompetitionPackage._records）

| 阶段 | 行为 |
|---|---|
| **完赛瞬间** | `CheckRecords` 遍历所有匹配 `项目+性别+组别(可空)` 且 `Time>0` 的记录条目；`FinalTime < record.Time` 写破纪录、`FinalTime == record.Time` 写平纪录；记录库为空则不比较 |
| **同步显示** | `result.RecordNote` 通过 `Broadcast` 同时推到主控泳道行（备注栏金色）、EXE 远程台、`race_control.html`、`display.html`（成绩位标红的反应时之外，备注金色显示标识） |
| **打印输出** | 每组成绩打印模板的成绩单元格后追加 `<span style='color:#b45309;font-weight:bold;'>WR</span>`（FINA 惯例）；"成绩与排名" Tab 的"纪录"列同步填值 |
| **裁判确认** | 按 **确认本组成绩**，`UpdateRecordsAfterConfirm` 把仍 `< record.Time` 的成绩写回 `SwimmingRecord`：刷新 `HolderName / HolderCountry / Time / TimeInSeconds / Date(今天) / Location(_competitionName)`，落盘到 CompetitionPackage |
| **平纪录不刷新** | `=` 前缀（平纪录）只标识，不更新原条目 |

### 11.4 破纪录统计表

> 主控参数设置弹窗（v2026.05）改为 `SizeToContent.Height` 自适应高度，去掉滚动条，不会因新增条目而显示不全。

#### 15.1.1 反应时(RT) 开 / 关详解

- **打开（默认）**：硬件检测到出发台触发后写入反应时；反应时低于阈值时自动标红"可疑起跳"（不再自动 DSQ）。
- **关闭**：所有反应时相关处理统一跳过——
  - `HandleFalseStartDetected` 直接 return
  - 出发台命中处理（普通+接力交接）：不写 `ReactionTime`、不判定可疑、不再标红
  - `result.StartingBlockTime` 不写入
  - 三端泳道行 `ReactionText` 与计时源详情面板：显示空 / "已关闭"
  - 广播 `reactionTime` 字段为空字符串

适用场景：训练记录 / 没有出发台硬件 / 不需要反应时统计的赛事。

#### 15.1.2 大屏翻屏时间详解

- 影响 `display.html` 总排名（多页超过 10 行时）的自动翻页周期
- 不影响其它显示模式（比赛视图 / 出发表 / 颁奖等）
- 实时生效：服务器广播后大屏立即采用新值，无需刷新页面

### 15.2 硬件连接

- 协议类型：串口 / UDP（局域网）/ TCP
- 串口参数：波特率（默认 9600）/ 串口号
- UDP / TCP：IP + 端口
- 测试 / 连接 / 断开 / 重连

```

                              ↓                                                   ↓

## 23. 常见问题

### 23.1 主服务器闪退或无法启动

- 查看是否被防病毒软件拦截
- 检查 .NET Framework 4.5+ 是否安装
- 端口 3002 是否被其它进程占用

### 23.2 客户端连不上主服务器

- 检查 IP 是否正确（用 `ipconfig` 确认主机 IP）
- 防火墙是否放行 `SwimmingScoreboard.exe`
- 主服务器右下角"连接状态"是否显示已启动 WebSocket

### 23.3 大屏卡顿

- v2026.04.29 已优化为分区刷新（100ms 滚动时间 + 250ms 廉价字段刷新 + DOM 缓存）
- 老版本浏览器（IE / 旧 Edge）渲染慢，建议 Chrome / 新 Edge

### 23.4 自动分组结果不符合 FINA 规则

- 请确认"项目类型"识别正确（短距 50/100/200 / 中长距离 400+ / 接力）
- 重新点 **预赛自动分组**（旧版数据手动调整后再点会重新执行 SW 3 算法）

### 23.5 已确认成绩消失

- 检查"赛程导航"是否仍显示 `[已完赛]`
- 在"成绩与排名"Tab 检查组次下拉是否包含该组
- 查看主控日志窗口是否有"赛程导航刷新失败 / 成绩与排名刷新失败"红色错误

### 23.6 接力队员重复登记

- v2026.04 已修复：相同 (队名 + 性别 + 项目) 的接力队会被识别为更新，不会增加重复行

### 23.7 计时器清零后大屏没切到欢迎画面

- 仅在"本赛次最后一组已确认 + 计时器清零"才会切欢迎画面
- 中间组的清零会自动切到下一组（仍显示比赛视图）

### 23.8 50/100/150 米触板打开时大屏闪现"出发反应时"

- v2026.04.29 已修复：分段计数只算"已记录到真实时间"的分段，PreCreateSplit 空占位不再触发反应时回放
