## 1. 系统概述

### 1.1 设计目标

本系统是一套**完整的游泳比赛管理与计时解决方案**，覆盖赛事全生命周期：

- **赛前**：赛事创建 / 运动员报名 / 自动或手动分组 / 秩序册编辑 / 纪录数据导入
- **赛中**：硬件计时实时联动 / 三端控制台同步 / 设备状态管理 / 大屏展示 / 媒体播放
- **赛后**：成绩确认与排名 / 多轮晋级 / 成绩册编辑 / 多格式文档输出（PDF / DOC / HTML）

### 1.2 系统组成

| 组件 | 说明 | 运行方式 |
|---|---|---|
| **主服务器** `SwimmingScoreboard.exe` | 赛事管理核心，连接计时硬件 | Windows 应用程序 |
| **远程计时控制台** `RemoteTimingControl.exe` | 独立 PC 上的比赛控制客户端 | Windows 应用程序 |
| **HTML 计时控制台** `race_control.html` | 浏览器端比赛控制（与 EXE 同等功能） | 浏览器 |
| **网页检录台** `checkin.html` | 检录台核对身份、打印检录单 | 浏览器 |
| **大屏显示** `display.html` | 观众大屏（项目/成绩/纪录/媒体）| 浏览器全屏 |
| **排名屏** `leaderboard.html` | 项目/团体/纪录排名展示 | 浏览器全屏 |
| **在线报名** `register.html` | 运动员/接力队自助报名 | 浏览器 |
| **注册工具** `RegistrationTool.exe` | 桌面端运动员注册客户端 | Windows 应用程序 |
| **自动化测试机器人** `test_bot.html` | 模拟报名/比赛流程，便于演示与压测 | 浏览器 |
| **计时模拟器** `TimingSimulator.exe` | 在没有实物硬件时调试协议 | Windows 应用程序 |
| **硬件参数调试机器人** `ParamDebugBot.exe` | 硬件双向调试 | Windows 应用程序 |

### 1.3 系统要求

- **操作系统**：Windows 7 / 8 / 10 / 11
- **运行环境**：.NET Framework 4.5（系统通常已预装）
- **网络**：局域网即可（主服务器与各客户端 / 大屏在同一子网）
- **端口**：`3002`（WebSocket）
- **浏览器**：Chrome / Edge（推荐）
- **硬件计时器**：通过串口 / UDP / TCP 任一方式连接（协议详见 §3）

## 2. 安装与启动

### 2.1 安装

1. 解压 `游泳赛事管理系统_安装包_<日期>.zip`
2. 双击 `Setup.exe`，选择安装目录（默认 `C:\SwimmingTimingSystem`）
3. 安装完成，桌面 / 开始菜单生成快捷方式

### 2.2 启动主服务器

1. 双击 **"游泳赛事管理主服务器"**
2. 首次启动设置管理员密码
3. 登录后进入主界面（标签页：赛事管理与报名 / 比赛控制 / 成绩与排名 / 纪录管理 / 比赛类型管理 / 文档编辑/输出/打印 / 系统日志与数据）

### 2.3 启动远程计时控制台（异机操作）

1. 双击 **"远程计时控制台"**
2. 顶部输入主服务器地址 `<IP>:3002`
3. 点击 **连接**

### 2.4 打开 HTML 控制台 / 大屏

- 浏览器输入 `http://<主服务器IP>:8080/race_control.html` 等（或直接打开本地 HTML 文件，首次输入服务器 IP）

### 2.5 防火墙

请允许 `SwimmingScoreboard.exe` 通过 Windows 防火墙；否则其它客户端将无法连接 3002 端口。

## 3. 系统架构与通讯

```
            ┌──────────────────────────────┐
            │       计时硬件控制器          │
            │   (串口 / UDP / TCP)         │
            └──────────┬───────────────────┘
                       │ 12 字节帧 F1 53 ...
                       │（双向同步：参数/设备/比赛距离）
        ┌──────────────▼──────────────────┐
        │    主服务器 SwimmingScoreboard   │
        │   (WebSocket :3002 / SQLite备份) │
        └──────────────┬──────────────────┘
                       │
        ┌──────────────┼─────────────────┬───────────────┐
        ▼              ▼                 ▼               ▼
   EXE 远程台    HTML 计时台       大屏 / 排名屏     在线报名 / 检录台
  (RemoteTimingControl)    (race_control.html)
```

### 3.1 三个计时控制台

主服务器 EXE / RemoteTimingControl EXE / race_control.html 三者**功能完全等价**，操作哪一台都通过 WebSocket 同步到其它两端。任意一端的"参数设置"修改会立即广播到全部端 + 硬件。

### 3.2 硬件计时协议

- 12 字节帧 `F1 53 ... F4`（详见 `硬件计时协议规范.html`）
- `0x42` 子码扩展（双向）：
  - `0x01-0x07` ：泳道关闭时间 / 出发台延迟 / 抢跳阈值 / 终点位置 等
  - `0x08`：左右盲表数量（高 4 位左 / 低 4 位右）
  - `0x10`：设备好/坏位图
  - `0x20`：比赛距离 + 左右触板次数

## 4. 比赛类型管理

`比赛类型管理` tab 用于配置 5 种基础维度：

| 维度 | 用途 | 默认值 |
|---|---|---|
| **比赛项目** | 报名 / 赛程的可选项目（50/100/200米…）| 标准游泳项目 |
| **组别** | 报名时分类（甲/乙/丙/丁/戊/己 等）| 6 个标准组 |
| **性别** | 男 / 女 / 混合 | 三个 |
| **赛次** | 预赛 / 半决赛 / 决赛 / 自定义 | 三个 |
| **组数** | 1 组 / 2 组 / … 的预设值 | 1-8 |

每个维度都有"**编辑** / **导入** / **导出** / **下载模板**" 4 个按钮，编辑器为弹窗 DataGrid（左侧自动序号 1, 2, 3 …，可增/删行），数据持久化于赛事档案。

> 注：组别与年龄无关，是人工分类（如奥运/全运的甲乙丙组），不再像旧版按 MinAge/MaxAge 自动判定。

## 6. 运动员管理

### 6.1 添加 / 修改 / 删除

- **添加运动员** 弹出对话框，可一次性填齐：姓名 / 性别 / 出生日期 / 身份证 / 代表队 / 单位简称 / 组别 / 联系电话 / 协会注册号 / 备注 / 报名项目 / 报名成绩 等
- **修改选中** / **删除选中**：支持多选（Ctrl/Shift），按 Delete 键也可删除
- **添加接力队**：支持 4 棒队员姓名 / 身份证 / 队号录入

### 6.2 筛选

顶部筛选器：项目 / 组别 / 性别 / 姓名（模糊）/ 号码（模糊）。状态修改后立即同步到 race control / 大屏。

### 6.3 号码区间设置

按代表队设置号码段（如 北京队 001-050）。新注册运动员自动在该队号码段内取号；也可手动指定。

### 6.4 CSV / Excel 批量

- **导入 CSV**：旧版扁平列表导入
- **导入 / 导出 / 下载模板**：详见 §7.3 分组表 Excel

## 7. 赛程与分组

### 7.1 自动 / 手动赛程生成

- **一键生成日程**：按"组别 → 性别 → 项目 → 赛次"维度合成场次，自动按距离 / 项目类型排序
- **手动赛次设置**：弹出 `PromptStagePlan` 让用户为每个项目自定义需要的赛次（仅决赛 / 预+决 / 预+半+决 / A决+B决 / 自定义）

### 7.2 自动 / 手动分组

- **自动生成分组**：按报名成绩蛇形分配
- **出场编排微调**：可拖动调整每道运动员，处理空泳道 / 临时加人

### 7.3 分组表 Excel 导入 / 导出（**新增**）

`赛事管理与报名 → 导入分组表 (Excel)` / `导出分组表 (Excel)` / `下载分组表 Excel 模板`

**Excel 工作簿含 3 个 sheet**：

1. **「分组明细」**（行式 / 主数据） 19 列：
   场次 / 比赛日期 / 时段 / 项目编号 / 组别 / 性别 / 项目 / 赛次 / 排序方式 / 组号 / 道次 / 参赛号 / 姓名 / 代表队 / 单位简称 / 报名成绩 / 出生年月 / 年龄 / 备注
2. **「分组表(网格)」**：仿全运会秩序册排版，每组 3 行（道次 / 姓名 / 代表队），接力额外 4 棒
3. **「填写说明」**：说明 + 完整项目清单 + 已注册代表队列表

**导入对话框** 含「**未注册时自动添加**」复选框（默认勾选）—— 当 Excel 中的姓名+代表队 在系统中未找到时，自动按 Excel 行新建运动员。

10 道泳池表头显示道次 `0~9`；8 道为 `1~8`。

### 7.4 晋级处理

`成绩与排名` tab 的 **晋级处理** 按钮调出 `PromotionQueryWindow`，支持：
- 查询当前赛次的总排名（不分组统一排名）
- 设置晋级人数（默认 16）
- 一键执行晋级 + 自动分组到下一赛次

## 8. 比赛控制

`比赛控制` tab 三栏布局：

```
左：赛程导航 + 计时源对比          中：泳道实时状态                 右：比赛控制按钮
```

### 8.1 泳道实时状态（左右对称布局）

每道一行，左右对称：

```
[T] 盲3 盲2 盲1 出发 触板 圈 │ 姓名+进度 │ 圈 触板 出发 盲1 盲2 盲3 [T]
```

- **左右盲表数量** 可独立设置 1-3，超出数量的圆点占位但隐藏（盲 1 / 出发 / 触板的位置始终固定）
- **T 按钮**：手动触板（左右两侧各一个，可单独启用）
- **进度条**：箭头方向跟随发令位置；显示运动员所在段
- 顶部条显示 **大屏主纪录**（WR / AR / NR / 省R / 市R 之一，可在记录管理中切换）

### 8.2 比赛控制按钮组

- **就位** / **发令** / **计时复位** / **确认成绩**
- **一键全开/全关泳道** / **重启** / **手动盲表成绩**
- **上一组 / 下一组**

### 8.3 显示控制（右侧）

向大屏推送：**出发表 / 比赛视图 / 组成绩 / 总排名 / 团体 / 颁奖 / 纪录 / 欢迎 / 成绩发布 / 图片/视频**

### 8.4 比赛流程

1. 选择项目 → 选组 → **就位**
2. **发令**（可由硬件触发或软件触发）
3. 触板/盲表/手动按钮接收 → 自动判定计时源
4. 完成 → 检查抢跳/犯规 → **确认成绩**
5. 一组完成可 **下一组**（已确认组返回时仍能看到成绩）

> **跨组成绩保留**：进入已确认组时，自动恢复完赛状态显示，不会因切组而丢失之前的成绩显示。

## 11. 文档编辑/输出/打印

`文档编辑/输出/打印` tab 把所有文档归类成 4 大块（每个文档为统一卡片）：

| 一、赛前文档 | 二、比赛成绩 | 三、排名与纪录 | 四、证书与奖状 |
|---|---|---|---|
| 秩序册 [编辑\|输出/打印] | 成绩册 [编辑\|输出/打印] | 团体成绩 | 奖状 |
| 竞赛日程 | 分组成绩 | 纪录报告 | 纪录证书 |
| 出发表 | 项目成绩 | | |
| | 分段计时报告 | | |

### 11.1 统一预览/输出窗口（**新增**）

每个"输出/打印"按钮弹出 `DocumentPreviewWindow`：
- **预览 tab**：内嵌浏览器渲染
- **编辑源码 tab**：可手改 HTML 后刷新预览
- 底部按钮：**刷新预览** / **在浏览器中打开** / **导出 PDF**（浏览器 → Microsoft Print to PDF）/ **导出 DOC** / **导出 HTML** / **打印**

文档生成时会自动归档到 `Documents/` 目录。

### 11.2 秩序册编辑器

`编辑秩序册` 按钮 → 7 选项卡弹窗：
1. **封面**：主标题 / 副标题 / 场地示意图
2. **前言/规程/通知**：3 个多行文本区
3. **重要活动日程**：日期/时间/活动/参与人员/地点
4. **训练日程**：日期/时间/地点
5. **竞赛人员**：职务/姓名（仲裁/技术官员）
6. **运动队人员**：每队的领队/教练/队医/工作人员（"从已报名队伍自动建表"按钮）
7. **附注/尾页**

任意空字段使用系统默认内容。秩序册可导出为 PDF/DOC/HTML。

### 11.3 成绩册编辑器

`编辑成绩册` 按钮 → 4 选项卡：
1. **封面/前言/尾页**
2. **体育道德风尚奖**：运动队 / 运动员 / 裁判员 三栏
3. **教练员/联合培养**：DataGrid，按号码批量编辑（"从已注册运动员自动建表"按钮）
4. **显示选项**：奖牌榜 / 风尚奖 / 破纪录统计 / 名次公告 / 成绩公告 / 分段成绩 / 成绩差 5+2 个开关

成绩册结构（参照全运会成绩册）：封面 → 目录 → 奖牌榜 → 风尚奖 → 破纪录统计表 → 名次公告（每项目决赛排名 + 教练员/联合培养单位）→ 成绩公告（每组逐运动员，可含分段成绩）→ 尾页。

### 11.4 「成绩差」列（**新增**）

打印 / 输出的所有"每组比赛成绩"都包含 **成绩差** 列，**适用所有赛次**（预赛 / 半决赛 / 决赛 …）：
- 第 1 名留空
- 其它道：`成绩差 = 当前道成绩 - 第1名成绩`，保留 2 位小数
- DSQ / DNS / DNF 行不计算

### 11.5 输出格式选择

- **PDF**：浏览器 Ctrl+P → 选择 "Microsoft Print to PDF" 打印机
- **DOC**：直接保存为 .doc，Word 可打开
- **HTML**：保留原始 HTML（可二次编辑）
- **直接打印**：调用系统打印对话框

## 12. 图片/视频大屏显示

`比赛控制 → 显示控制` 新增 **「图片 / 视频」** 按钮 → 弹出对话框：

1. **选择文件**：PNG / JPG / BMP / GIF / WEBP / MP4 / WEBM / OGG
2. **缩放方式**：保持比例（contain）/ 填满屏幕（cover）/ 拉伸（fill）
3. **视频选项**：循环播放 / 静音 / 自动播放
4. **发送到大屏** → base64 推送到所有 `display.html` 客户端，全屏播放
5. **停止显示** → 回到比赛视图

> 文件大于 60 MB 时会弹出确认（base64 编码可能耗时）。

## 13. 系统日志与数据管理

`系统日志与数据` tab：
- **运行日志**（左）：实时滚动
- **存档管理**（右）：
  - **加载 / 删除选中存档**
  - **清除当前库数据**（含确认）
  - **删除全部纪录**（含确认）
  - **强制保存**
  - **查询本机 IP 地址**
  - **修改密码**

赛事档案存于 `Database/<赛事名>.json`，包含运动员 / 接力 / 赛程 / 纪录 / 团体分 / 秩序册 / 成绩册 / 显示纪录设置等所有数据。

## 14. 参数设置与硬件连接

`比赛控制 → 参数设置` 按钮（齿轮图标）弹出（**自适应高度，无滚动条**）：

### 14.1 计时参数

- 泳道关闭时间 / 出发台关闭延迟 / 成绩确认关闭延迟 / 抢跳判定阈值 / 分段成绩显示时长 / 第1名成绩停留时间
- **终点位置**：左端 / 右端

### 14.2 设备状态管理 / 手动按键管理

- 设备好/坏：每道每个设备（触板/出发台/盲表1/2/3）独立标记，影响计时源仲裁
- 手动按键：每道左右T按钮启用/禁用

### 14.3 左右盲表数量设置（**新增**）

弹出子窗，左右各 1 个 1/2/3 下拉：
- 修改后立即同步到三个计时控制台 + 大屏 + 硬件（0x42 / 0x08 子码）
- 隐藏多余圆点时**保留位置**，让"盲 1 / 出发台 / 触板"的位置永远固定
- 三端 + 大屏的圆点 / 表头标签 / 表头列**完全对称**

### 14.4 计时硬件直连

支持三种模式：
- **串口** (COM3 等)
- **UDP**（接收端口 5002）
- **TCP 客户端**（127.0.0.1:5555）

主服务器与硬件双向同步：参数变更（任一方）即推送给另一方。

## 22. 常见问题

**Q1：HTML 控制台连不上服务器？**
A：检查主服务器是否运行、防火墙是否放行 3002 端口、客户端与服务器是否同子网；或者点击控制台连接框旁的"换服务器"重新输入 IP。

**Q2：硬件计时器无信号？**
A：先在"参数设置 → 计时硬件直连"确认连接方式与端口；可用 `ParamDebugBot.exe` 单独调试帧通讯。

**Q3：盲表数量改了但 race_control 上方的"盲 1/2/3"标签没变？**
A：需要刷新浏览器或等待下一次广播；表头按 `applyBlindWatchVisibility()` 自动重画。

**Q4：从主控切到第 2 组确认后，第 1 组成绩看不到了？**
A：已修复 —— 进入已确认组时自动恢复完赛状态显示。如仍异常，请关闭主服务器后重新启动以加载最新代码。

**Q5：导出秩序册 / 成绩册时列内容未显示？**
A：确认秩序册/成绩册编辑器中相应 tab 的字段已填，或 `显示选项` tab 中对应章节开关已勾选。

**Q6：导出 PDF 时浏览器没自动选 PDF 打印机？**
A：浏览器 Ctrl+P 后在"打印机"下拉中选 "Microsoft Print to PDF"（Win10/11 自带）或第三方 PDF 打印机。

**Q7：分组表 Excel 导入未匹配的运动员怎么处理？**
A：导入对话框勾选 ☑ "未注册时自动添加"，未匹配的行将按 Excel 内容自动新建运动员。

**Q8：纪录显示标签需要从 WR 改为 AR / NR / 市记录…**
A：纪录管理 tab → 右上角 **大屏显示记录** → 选择或新增项 → 确定，三端 + 大屏立即同步。
